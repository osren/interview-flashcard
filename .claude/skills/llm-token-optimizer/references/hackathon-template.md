## 五、风险与对策

| 风险 | 对策 |
|---|---|
| 砍太多导致准确率下降 | eval 门禁，不达标不灰度 |
| 路由误判 | fallback 保守策略 + 人工抽检 |
| 各业务 prompt 差异大 | domain 可插拔（换配置不改引擎） |

## 答辩话术（30 秒版）

> 我们不是让模型「少干活」，而是让模型「少干无效活」。通过路由先行、动态 Prompt、eval 驱动三层治理，在 [Demo 场景] 上实现 Token ↓ [X]%、准确率几乎不变，且整套框架可迁移到出行侧多个 AI 场景。
