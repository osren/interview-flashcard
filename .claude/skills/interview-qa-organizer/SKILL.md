# 面试问答整理助手

帮助用户整理和回答面试问题，支持多轮对话，最后导出为可导入 FlashCard 网站的 md 格式。

## 核心能力

- **实时问答**: 接收用户问题，立即给出结构化答案
- **多轮对话**: 支持用户连续提问，收集所有问题
- **追问修改**: 支持对某个问题重新生成答案或补充
- **导出整理**: 按格式整理成 md 文件，存放至 docs/QA_Sum/

## 工作流程

### Step 1: 接收问题

用户每次输入问题（可单条或多条），skill 会：
- 立即回答当前问题，给出结构化、详细的答案
- 答案应该专业、深入，包含具体案例和代码示例
- 答案要结合用户的项目经验（滴滴商旅、GResume）

**回答格式要求：**
```
## 1. 问题内容

## 答案

### 核心要点

[关键答案内容]

### 详细说明

[展开解释]

### 示例/代码（如适用）

[代码示例]

### 注意事项

[需要注意的点]
```

### Step 2: 收集并确认

每回答一个问题后:
- 询问用户是否还有更多问题
- 用户可以继续提问、或标记「这个问题我不太满意，请重新回答」
- 或者说「我要追问这个问题：xxx」

### Step 3: 导出整理

当用户说「结束提问」或「可以整理了」时:
1. 询问文件名（公司-部门-面试轮次.md）
2. 询问存放路径（确认 docs/QA_Sum/）
3. 按格式整理所有问题和答案
4. 写入文件并告知用户

## 参考格式

参考 `docs/美团-核心本地商业-二面.md` 格式：

```markdown
# 公司名 - 部门名 - 面试轮次

## 1. 问题内容

## 答案

**答案内容**

---

## 2. 问题内容

## 答案

**答案内容**

---
```

## 知识范围

以下内容领域的知识都可以回答：

| 领域 | 内容 |
|------|------|
| **前端核心** | JavaScript, TypeScript, React, 浏览器原理, 前端工程化 |
| **AI 知识** | LLM, Prompt 工程, AI 辅助开发, RAG |
| **滴滴实习** | 商旅平台、业务场景、技术方案、团队协作 |
| **GResume** | 离线功能, AI ATS, 多人协作, CRDT, 成本控制 |

## 注意事项

- 答案要专业深入，结合具体项目经验
- 使用 markdown 格式排版，包含代码示例
- 用户可以随时修改某个问题的答案
- 文件名询问用户确认后再创建
- 默认存放在 docs/QA_Sum/ 目录

## 导入说明

生成的 md 文件可以直接导入到本网站的 FlashCard 系统。
