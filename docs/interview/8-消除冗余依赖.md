# 消除冗余依赖的方法

## 一、依赖分析与管理

### 1. 依赖审计

- 定期使用 `npm audit` 或 `depcheck` 分析项目依赖，识别未使用或重复的包
- 本项目使用 Vite 2.1.5，依赖 tree-shaking 能力减少冗余

### 2. 版本锁定

- 关键依赖锁定版本（如 `@didi/dajax` 必须为 3.0.1，升级会导致请求头 content-type 被覆盖的问题）
- 使用 package-lock.json 或 yarn.lock 锁定传递依赖

## 二、代码分割策略

### 1. 路由级分割

- 使用 `loadable()` 按需加载57个业务页面，首屏只加载必要代码
- 配置示例（Routes.jsx）：

```javascript
'/travel_config_general': loadable(() => import(`./pages/travel_config`), {
  fallback: <div>Loading...</div>
}),
```

### 2. 组件级分割

- 业务子页面独立打包，如 `plane_policy/page/plane_cabin`
- 弹窗、抽屉等非首屏组件采用动态导入

## 三、依赖优化手段

### 1. 路径别名配置

Vite 配置路径别名减少相对路径复杂度：

```javascript
alias: [
  { find: 'Services', replacement: '/src/services/' },
  { find: 'Components', replacement: '/src/components/' },
  { find: 'Utils', replacement: '/src/utils/' },
]
```

### 2. 统一封装

- 请求层统一封装（`src/utils/request.js`），避免重复封装 axios
- 工具函数统一管理（`src/utils/util.js`），避免重复实现

### 3. 构建优化

- 生产环境使用 Webpack 打包，启用 Tree Shaking
- Ant Design 按需引入（babel-plugin-import）
- 将大型库（如 moment）替换为轻量替代（如 dayjs）

## 四、检测与约束

### 1. ESLint 规则约束

- 项目禁用了一些规则以适配实际需求，但核心规则仍约束无用代码
- 检测未使用的导入、变量等

### 2. 代码审查

- 合并前审查依赖变更，确保不引入冗余依赖
