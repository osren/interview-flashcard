# 弱网场景路由分割、预加载及构建优化

## 一、路由分割（Code Splitting）

### 1. 实现方式

本项目使用 `@loadable/component` 实现按需加载，这是 React 官方推荐的分割方案。配置于 `Routes.jsx`，每个路由对应一个独立的 chunk：

```javascript
const ROUTETOCOMPONENTS = {
  '/ebk': loadable(() => import(`./pages/ebk`), {
    fallback: <div>Loading...</div>
  }),
  '/plane_policy': loadable(() => import(`./pages/plane_policy`), {
    fallback: <div>Loading...</div>
  }),
  // ... 共57个页面
};
```

### 2. 分割粒度

- **页面级分割**：每个业务模块独立 chunk
- **子页面级分割**：如 `plane_policy/page/` 下的多个子页面独立加载

### 3. 弱网收益

- 首屏 JS 体积从整体压缩到按需加载，弱网下首屏时间大幅降低
- 用户点击导航时才加载对应 chunk，减少初始等待

## 二、预加载策略

### 1. 骨架屏 fallback

当前使用简单的 Loading 占位符，实际可优化为骨架屏：

```javascript
fallback: <div className="skeleton-container">...</div>
```

骨架屏提供视觉反馈，降低用户感知等待时间。

### 2. 资源预加载

可使用 Webpack magic comments 实现预加载：

```javascript
import(/* webpackPrefetch: true */ './pages/travel_config')
```

浏览器空闲时预取后续可能访问的 chunk。

### 3. 预判加载

- 基于用户行为预判（如鼠标悬停导航）触发加载
- 路由守卫中根据权限预加载相关模块

## 三、构建优化

### 1. 开发环境优化

- Vite 提供基于 ESM 的 HMR，热更新时间 <100ms
- 代理配置支持本地 mock 和跨域调试

### 2. 生产环境优化

- 使用 Webpack 构建（`npm run webpack`）
- 代码分割：每个路由独立 chunk
- 代码压缩：TerserPlugin 压缩 JS
- CSS 提取：mini-css-extract-plugin 提取 CSS

### 3. 体积优化

- Ant Design 按需引入（实际项目中全量引入，可优化）
- moment.js 替换为 dayjs（当前项目使用 moment 2.29.1，可优化）
- lodash 按需引入（当前全量引入，可优化为 lodash-es）

### 4. 弱网专项优化

- **HTTP/2 多路复用**：减少 TCP 连接数，降低 RTT
- **Gzip/Brotli 压缩**：传输体积减少 70%+
- **CDN 加速**：静态资源部署至 CDN，就近访问
- **Service Worker 缓存**：离线可用，二次访问秒开
- **资源优先级**：critical CSS 内联，非关键资源 defer/async
