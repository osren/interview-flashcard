## 技术指标

### 首屏加载 TTI (Time to Interactive)

**定义**：页面可交互时间，即页面完成首次加载并能够响应用户操作的时间点。

**计算方法**：
```
TTI = DOMContentLoaded 事件时间 + 5秒内没有长任务 + 页面完成至少5秒的网络请求
```

**衡量标准**：
- 优秀：< 3秒
- 良好：3-5秒
- 及格：5-8秒
- 需优化：> 8秒

---

### 白屏时间 FP (First Paint)

**定义**：页面首次渲染的时间点，即浏览器开始绘制任何可视内容的时间。

**计算方法**：
```javascript
// 使用 Performance API
const fp = performance.getEntriesByName('first-paint')[0]?.startTime;
```

**衡量标准**：
- 优秀：< 1秒
- 良好：1-2秒
- 及格：2-3秒
- 需优化：> 3秒

---

### 操作设备统计

**定义**：记录用户访问页面的设备类型、屏幕尺寸、操作系统、浏览器等维度分布。

**计算方法**：
```javascript
const deviceInfo = {
  device: navigator.platform,
  screen: `${window.screen.width}x${window.screen.height}`,
  pixelRatio: window.devicePixelRatio,
  os: navigator.userAgent
};
```

**衡量标准**：按业务场景设定目标值，重点关注主流设备（iOS/Android 高端机）的覆盖率和体验。

---

### 启动耗时

**定义**：从用户启动应用到页面首次渲染完成的时间。

**计算方法**：
```
启动耗时 = 应用启动时间 + 页面渲染时间
```

**衡量标准**：
- 冷启动：< 3秒（优秀），3-5秒（良好），> 5秒（需优化）
- 热启动：< 1秒（优秀），1-2秒（良好），> 2秒（需优化）

---

### 启动成功率

**定义**：启动过程中无错误、无卡死、正常加载完成的比例。

**计算方法**：
```
启动成功率 = (启动成功次数 / 总启动次数) × 100%
```

**衡量标准**：
- 优秀：≥ 99.5%
- 良好：99%-99.5%
- 及格：95%-99%
- 需优化：< 95%

---

### 首屏页面渲染耗时

**定义**：页面首屏内容完全渲染的时间，包括首屏图片、文本、布局等。

**计算方法**：
```javascript
// 监控首屏渲染完成时间
window.addEventListener('load', () => {
  const paint = performance.getEntriesByType('paint');
  const firstContentfulPaint = paint.find(e => e.name === 'first-contentful-paint');
});
```

**衡量标准**：
- 优秀：< 1.5秒
- 良好：1.5-2.5秒
- 及格：2.5-4秒
- 需优化：> 4秒

---

### 页面渲染耗时

**定义**：页面从开始渲染到完成渲染的时间。

**计算方法**：
```
页面渲染耗时 = DOMContentLoaded 事件时间 - 页面开始加载时间
```

**衡量标准**：
- 单页应用：< 2秒（优秀）
- 复杂页面：< 3秒（优秀）

---

### 页面渲染成功率

**定义**：页面渲染过程中无错误、正常展示的比例。

**计算方法**：
```
页面渲染成功率 = (渲染成功次数 / 总渲染次数) × 100%
```

**衡量标准**：
- 优秀：≥ 99%
- 良好：97%-99%
- 及格：95%-97%
- 需优化：< 95%

---

### 完全加载耗时 (Load Time)

**定义**：页面所有资源（包括图片、脚本、样式表等）全部加载完成的时间。

**计算方法**：
```javascript
window.addEventListener('load', () => {
  const loadTime = performance.timing.loadEventEnd - performance.timing.navigationStart;
});
```

**衡量标准**：
- 优秀：< 3秒
- 良好：3-5秒
- 及格：5-8秒
- 需优化：> 8秒

---

### 核心页面白屏率

**定义**：核心页面在加载过程中出现白屏现象的比例。

**计算方法**：
```
核心页面白屏率 = (核心页面白屏次数 / 核心页面访问总次数) × 100%
```

白屏判定：页面加载超过指定时间（如 5 秒）仍未有任何内容渲染。

**衡量标准**：
- 优秀：< 0.1%
- 良好：0.1%-0.5%
- 及格：0.5%-1%
- 需优化：> 1%

---

### 首次内容渲染 FCP (First Contentful Paint)

**定义**：页面首次渲染任何内容（文本、图片、SVG 或 canvas）的时间。

**计算方法**：
```javascript
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.name === 'first-contentful-paint') {
      console.log('FCP:', entry.startTime);
    }
  }
});
observer.observe({ type: 'paint', buffered: true });
```

**衡量标准**：
- 优秀：< 1.2秒
- 良好：1.2-1.8秒
- 及格：1.8-3秒
- 需优化：> 3秒

---

### 最大内容绘制 LCP (Largest Contentful Paint)

**定义**：页面视口内最大可见元素（图片、视频或文本块）渲染的时间。

**计算方法**：
```javascript
const observer = new PerformanceObserver((list) => {
  const entries = list.getEntries();
  const lastEntry = entries[entries.length - 1];
  console.log('LCP:', lastEntry.startTime);
});
observer.observe({ type: 'largest-contentful-paint', buffered: true });
```

**衡量标准**：
- 优秀：< 2.5秒
- 良好：2.5-4秒
- 及格：4-6秒
- 需优化：> 6秒

---

### 进入方式占比

**定义**：用户进入页面的来源渠道分布，如直接访问、搜索引擎、外部链接等。

**计算方法**：
```javascript
const referrer = document.referrer;
// 按来源分类统计
const source = referrer.includes('google') ? '搜索引擎'
  : referrer === '' ? '直接访问'
  : '外部链接';
```

**衡量标准**：根据业务目标设定各渠道占比目标，重点关注转化率高的渠道。

---

## 综合性能评估

### Web Vitals 综合评分

| 指标 | 优秀 | 良好 | 及格 | 需优化 |
|------|------|------|------|--------|
| LCP | < 2.5s | 2.5-4s | 4-6s | > 6s |
| FCP | < 1.2s | 1.2-1.8s | 1.8-3s | > 3s |
| TTI | < 3s | 3-5s | 5-8s | > 8s |
| CLS | < 0.1 | 0.1-0.25 | 0.25-0.5 | > 0.5 |

### 性能监控建议

1. **数据采集**：通过 Performance API、PerformanceObserver 实时采集
2. **采样率**：生产环境建议 5%-10% 采样，避免性能开销
3. **上报策略**：结合错误监控，关键指标实时上报，长任务异步上报
4. **阈值告警**：设置 P95/P99 分位数告警，及时发现性能退化
