# Proposal: MPX 基础语法闪卡补充

## 需求背景

### 问题陈述

当前 `src/data/mpx/mpx.ts` 中的 MPX 卡片是从 `docs/Mpx/*.md` 文档自动解析生成的，解析逻辑为：按 `---` 分隔文档，提取 `## N. 标题` 模式的内容作为卡片。这种方式生成的卡片存在以下问题：

1. **缺少结构化字段** — 无 `category`（分类）、`difficulty`（难度等级）、`codeExample`（代码示例）等关键字段
2. **问答格式不匹配** — 文档内容是叙述性教程，而非问答式闪卡，导致 question 字段过长的同时 answer 缺乏针对性
3. **基础语法覆盖不足** — 文档侧重完整语法参考，缺少针对"基础语法模块"的专项闪卡集合

### 用户角色

- **候选人**：准备 MPX 小程序框架相关的面试
- **需求**：通过闪卡练习掌握 MPX 基础语法核心概念

### 关键旅程

1. 用户进入 `/algorithms` 或 `/core` 页面，选择 MPX 模块
2. 用户浏览"基础语法"章节的闪卡列表
3. 用户点击卡片查看问题，思考后翻转查看答案
4. 用户标记卡片状态（forgotten/fuzzy/mastered）推进学习

### 影响范围

- **新增**：`src/data/mpx/mpx.ts` 中补充结构化闪卡
- **修改**：可能需要调整 `mpxChapters` 章节配置以包含新的基础语法章节
- **不影响**：其他模块（core/projects/algorithms）的现有卡片

### 风险

- 新卡片内容需与现有文档内容保持一致，避免信息冲突
- 需确保新增卡片的 `id` 与现有卡片不冲突

### 待澄清项

1. 新增的基础语法卡片是否需要单独作为一个 chapter，还是合并到现有 `mpx-guide` 章节？
2. 卡片数量目标是多少？（建议每个语法点 3-5 张卡）

### 信息来源追溯

- 用户需求：补充 MPX 模块卡片内容，注重基础语法（参考 https://mpxjs.cn/guide/basic/intro.html）
- 现有数据：`docs/Mpx/mpx-guide.md`（1780 行完整 MPX 语法指南）
- 闪卡结构：`src/types/card.ts` 中定义的 `FlashCard` 接口
- 参考格式：`src/data/core/javascript.ts` 中的结构化闪卡示例
