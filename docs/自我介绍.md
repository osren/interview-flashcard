面试官你好，我叫谭成，一名研二学生，就读于某高校计算机专业，目前课程已基本修完，主要精力放在论文研究和实习上。我的求职方向是前端开发工程师。

先说技能掌握吧。我主要从事前端开发工作，熟练掌握 React、TypeScript、Vue 等主流前端框架，在状态管理方面有 Redux 和 Zustand 的使用经验，UI 组件库主要用的是 Ant Design 和 Tailwind CSS，工程化方面熟悉 Vite 和 Webpack 的配置，另外还接触过 Supabase 做后端服务，以及 Claude Code 进行 AI 辅助开发。

---

接下来说说我的实习经历。我从 2025 年开始在滴滴企业版实习，主要参与商旅系统的开发工作。在滴滴的实习主要围绕三个方面展开。

第一块是遗留系统重构。大家都知道，老项目最难维护的就是历史代码，滴滴的机票政策模块是 55 个业务模块中最复杂的，单文件超过 2000 行代码，运行了超过 5 年。国内和国际两套代码分离，重复开发，维护成本很高。我参与将这个模块从 jQuery 加硬编码的方式转变为配置驱动开发，通过设计 JSON Schema 动态表单，把业务逻辑抽离为配置，让新增字段不需要改代码，只要改配置就行。同时我也负责了一些通用组件的封装，比如下拉选择器、表格组件等，提升了团队的开发效率。

第二块是弱网性能优化。滴滴商旅的用户经常在出差途中使用系统，网络环境不稳定是常态。我针对这个场景做了路由分割和预加载优化，使用 @loadable/component 实现按需加载，配合鼠标悬停预加载和空闲时间预加载策略，最终将 FCP 从 3.2 秒优化到 1.8 秒，首屏 JS 体积减少了 45%，弱网环境下的白屏时间减少了 50%。

第三块是 AI 工程化转型，这也是我觉得最有收获的一部分。2024 年是 AI 爆发的一年，滴滴内部也在探索 AI 辅助开发的最佳实践。我参与了团队内部的 OpenSpec 规范制定工作，负责定义组件描述的 JSON Schema，编写 AI 交互的 Prompt，以及设计代码质量的检测规则。同时我也负责维护团队的 CLAUDE.md 项目知识库，让 AI 能够更好地理解项目背景和技术约束。在这个过程中，我推动了一些 Claude Code Agent 辅助开发工作流的落地，包括需求分析、代码生成、代码审查等环节，最终帮助团队在大需求开发周期上缩短了 30%。

---

除了实习经历之外，我还独立开发了一个叫 GResume 的智能简历平台，这是一个开源项目，目前已经在 GitHub 上可以看到。

这个项目源于我自己的求职需求。当时我发现市面上现有的简历工具要么无法离线编辑、要么数据不安全、要么缺乏 AI 优化能力，正好我也在学习 CRDT 相关技术，就想着做一个来解决这些痛点。

在技术实现上，我采用了离线优先的架构，用 IndexedDB 做本地存储，用 CRDT（基于 Automerge 库）来解决多人协作时的冲突问题，同时接入了 Supabase 做云端同步。也就是说，用户在地铁里、飞机上都可以正常编辑简历，等网络恢复后会自动同步到云端，不用担心数据丢失。

另外我也集成了 AI ATS 评分功能，接入 DeepSeek LLM 来分析简历与岗位 JD 的匹配度，给出可视化的优化建议，并且支持流式响应，用户可以实时看到 AI 的分析过程。

在性能方面，我做了路由级代码分割，用 Vite 的 manualChunks 将 TipTap 编辑器、Automerge 核心库等大模块单独打包，首屏只加载必要的代码。同时实现了乐观 UI 的更新策略，让用户的每次操作都能立即得到响应，配合光标节流批处理，将同步请求减少了 80%。
