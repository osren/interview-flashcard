## 12. 为什么选择 Automerge 而不是 Yjs？选型考量是什么？

## 答案

### 技术选型对比

| 维度 | Automerge | Yjs |
|------|-----------|-----|
| **数据结构** | 任意 JS 对象 + 嵌套 | 文档树顶层只能 Map/Array |
| **类型系统** | 原生支持对象、嵌套 | 需要手动注册 Y.Map/Y.Array |
| **同步协议** | 自动生成 Binary Delta | 需要手动计算 |
| **存储格式** | Base64 + 可选压缩 | Binary |
| **离线支持** | 友好 | 一般 |
| **依赖大小** | ~200KB | ~30KB 核心 |
| **生态** | Tiptap/ProseMirror 官方 | 更广（白板、表格等） |

### 本项目的核心考量

**1. 数据模型天然匹配**

简历是嵌套的 JSON 结构：

```
Automerge 原生支持：
type Resume = {
  basics: { name: string; email: string }
  education: EducationItem[]
  experience: ExperienceItem[]
}

Yjs 需要逐层包装：
const doc = new Y.Doc()
const basics = doc.getMap('basics')
const education = doc.getArray('education')
```

**2. Tiptap 编辑器官方集成**

项目使用 Tiptap（基于 ProseMirror），Automerge 是官方协作方案：

```
import { Collaboration } from '@tiptap/extension-collaboration'
import { AutomergeProvider } from '@automerge/automerge-repo'

Yjs 需通过第三方 provider 自行组装
```

**3. Repo 架构简化协作**

```
 Automerge Repo 自带网络适配器：
const repo = getAutomergeRepo(resumeId)
const handle = await repo.find(resumeId)
const collaboration = new CollaborationSessionManager(repo, handle)

Yjs 需要自己组装：NetworkProvider + Awareness + SyncProtocol
```

**4. 离线持久化更自然**

```
Automerge 保存/恢复：
const binary = doc.export()
const newDoc = new Doc().import(binary)
```

### 总结

```
| 场景                      | 推荐            |
|---------------------------|----------------|
| JSON 文档协作 + Tiptap 编辑器 | Automerge ✓   |
| 纯文本实时协作（白板、表格）     | Yjs           |
| 需要小体积、丰富生态         | Yjs           |

本项目是简历编辑器，核心是结构化 JSON + 富文本编辑，
Automerge 与 Tiptap 的官方集成是主要决策点。
```

## 13. TipTap 富文本编辑器怎样设计接入系统的？

## 答案

### 技术选型

- **基于 ProseMirror**：Tiptap 是 ProseMirror 的 React 封装
- **版本**：Tiptap 3
- **集成方式**：独立封装组件 + 自定义 Hook

### 架构设计

**组件分层：**

```
┌─────────────────────────────────────────┐
│  pages/resume/editor/components/forms/   │  ← 业务表单层
│  (WorkExperienceForm, ProjectForm...)    │
├─────────────────────────────────────────┤
│  components/tiptap-templates/          │  ← 编辑器模板
│  SimpleEditor                       │
├─────────────────────────────────────────┤
│  components/tiptap-ui/              │  ← UI 控制层
│  (BoldButton, HeadingButton...)     │
├─────────────────────────────────────────┤
│  components/tiptap-node/             │  ← 扩展节点
│  (ImageUploadNode)                 │
├─────────────────────────────────────────┤
│  hooks/use-tiptap-editor.ts          │  ← 核心 Hook
└───���─────────────────────────────────────┘
```

### 核心实现

**1. 自定义 Hook 统一管理编辑器实例**

```typescript
// src/hooks/use-tiptap-editor.ts
export function useTiptapEditor(providedEditor?: Editor | null) {
  // 优先使用显式传入的 editor，否则使用上下文中的
  const mainEditor = providedEditor || coreEditor

  // 返回 editor 状态和 canCommand 能力
  return {
    editor: context.editor,
    editorState: context.editor.state,
    canCommand: context.editor.can,
  }
}
```

**2. 业务表单接入 SimpleEditor**

```typescript
// src/pages/resume/editor/components/forms/WorkExperienceForm.tsx
import { SimpleEditor } from '@/components/tiptap-templates/simple/simple-editor'

function WorkExperienceForm() {
  return (
    <Form>
      {/* 工作描述字段使用 Tiptap */}
      <FormField name="workInfo" render={({ field }) => (
        <SimpleEditor
          content={field.value}
          onChange={(editor) => field.onChange(editor.getHTML())}
        />
      )} />
    </Form>
  )
}
```

**3. SimpleEditor 核心配置**

```typescript
// src/components/tiptap-templates/simple/simple-editor.tsx
const editor = useEditor({
  extensions: [
    StarterKit.configure({
      horizontalRule: false,
      link: { openOnClick: false },
    }),
    TextAlign.configure({ types: ['heading', 'paragraph'] }),
    TaskList,
    Highlight.configure({ multicolor: true }),
    Image,
    ImageUploadNode.configure({
      accept: 'image/*',
      maxSize: MAX_FILE_SIZE,
      limit: 2,
      upload: handleImageUpload,
    }),
  ],
  content,
})
```

**4. 扩展能力**

| 扩展 | 功能 |
|------|------|
| TextAlign | 文本对齐 |
| TaskList | 任务列表 |
| Highlight | 背景高亮 |
| Image | 图片插入 |
| ImageUploadNode | 图片上传 |
| Superscript/Subscript | 上标/下标 |
| Typography | 排版优化 |

### 总结

```
设计要点：
1. 组件分层：业务表单 → 编辑器模板 → UI 控制 → 扩展节点
2. Hook 封装：统一管理 editor 实例和状态
3. 外部控制：外部 content props 可控制编辑器内容
4. 事件回调：onChange 回调通知父组件变化
```
