# 项目复盘卡片生成指南

## 工作流程

### 1. 准备素材

请提供以下信息：

#### A. 项目本地路径
项目代码所在的文件夹路径（包含完整源码）

#### B. 简历对应描述
从简历中复制该项目相关的经历/项目描述

---

### 2. 我会做什么

1. **阅读项目源码** — 理解项目架构、技术栈、核心实现
2. **对照简历描述** — 找出简历中提到的亮点/难点
3. **生成问题清单** — 约 30 个深度问题，覆盖：
   - 技术选型（为什么用这个技术？）
   - 架构设计（整体设计思路？）
   - 难点攻坚（遇到了什么困难？怎么解决的？）
   - 性能优化（做了什么优化？效果如何？）
   - 工程化（团队协作？CI/CD？）
   - 业务理解（解决了什么业务问题？）
4. **生成卡片代码** — 直接追加到 `src/data/projects/` 对应文件

---

### 3. 卡片格式模板

```typescript
{
  id: '项目名-类别-序号',
  module: 'projects',
  chapterId: '项目名(英文)',
  category: '问题类别',
  question: '面试问题',
  answer: `## 回答标题

### 要点1
详细说明...

### 要点2
详细说明...

| 对比项 | A | B |
|--------|---|---|
| 特性1 | ✅ | ❌ |

关键术语加粗 **重点**`,
  tags: ['项目名', '技术栈', '类别'],
  status: 'unvisited',
  difficulty: 'hard',
},
```

---

### 4. 难度分级标准

| 难度 | 适用问题类型 |
|------|-------------|
| easy | 基础概念、业务背景、技术选型理由 |
| medium | 架构设计、性能优化、团队协作 |
| hard | 难点攻坚、复杂问题、深度原理 |

---

### 5. 类别建议

根据简历内容自定义，常见类别：

- `架构设计` — 系统设计、模块划分
- `技术选型` — 为什么用XX技术
- `性能优化` — 做了哪些优化、效果如何
- `难点攻坚` — 最难的问题、解决方案
- `工程化` — CI/CD、代码质量、团队协作
- `业务理解` — 解决了什么问题、价值是什么
- `AI集成` — AI相关功能（如有）
- `数据同步` — 离线、实时同步方案
- `用户体验` — 交互、可用性

---

## 示例：滴滴项目问题方向

```
技术栈：React/Vue、TypeScript、Webpack/Vite、SSG/SSR
业务：企业商旅系统
亮点：AI工程化（SSD/OpenSpec）、性能优化（弱网）、架构重构

可能的问题方向：
1. SSD规范驱动开发是什么？
2. 弱网环境下如何优化FCP？
3. 遗留系统重构的策略？
4. 如何推动团队AI转型？
5. SSG/SSR各自适用场景？
```
