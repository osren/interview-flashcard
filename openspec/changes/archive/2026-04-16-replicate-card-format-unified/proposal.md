# Proposal: 统一面经问答卡片格式到其他模块

## 问题背景

当前项目中存在两套不同的卡片系统：

1. **Interview 模块**（面经）：功能丰富 - 题号选择、导入导出、新增问题、上下题切换、答案编辑
2. **Core/Projects/Algorithms 模块**：使用基础 FlashCard 组件，功能简单

用户希望将面经模块的问答卡片格式复用到其他模块，统一各模块的使用体验。

## 当前实现差异

| 功能 | Interview 模块 | Core/Projects/Algorithms |
|------|----------------|--------------------------|
| 题号选择 | ✅ 题号按钮弹出选择弹窗 | ❌ 仅显示当前进度 |
| 导入导出 | ✅ ImportExportModal | ❌ 固定数据文件 |
| 新增问题 | ✅ 添加问题弹窗 | ❌ 静态数据 |
| 上下题切换 | ✅ 上一题/下一题按钮 | ✅ 基础导航 |
| 答案编辑 | ✅ MDEditor 即时编辑 | ✅ 保存到 modifiedCards |

## 目标

1. Core 模块支持动态导入导出问题
2. Projects 模块支持动态添加/编辑问题
3. Algorithms 模块支持动态问题管理
4. 统一各模块的交互体验

## 非目标

- 不修改 Interview 模块现有功能
- 不改变静态数据的默认加载逻辑
- 不添加后端服务，数据保持在 localStorage
