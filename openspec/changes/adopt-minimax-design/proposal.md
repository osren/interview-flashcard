# Proposal: 全局采用 MiniMax 设计风格

## 问题背景

当前项目（InterviewFlash）的视觉设计与 MiniMax 设计风格存在差异。用户希望全局采用 DESIGN.md 中定义的 MiniMax 设计风格进行改造。

## MiniMax 设计风格核心要点

### 1. 视觉主题
- 白色为主背景（#ffffff），彩色产品卡片作为视觉锚点
- 多字体系统：DM Sans（UI）、Outfit（展示）、Poppins（中等标题）、Roboto（技术数据）
- 品牌蓝色谱：#1456f0 到 #3b82f6
- 品牌粉色（#ea5ec1）作为辅助点缀

### 2. 色彩系统
- 主背景：纯白 #ffffff
- 主文本：近黑 #222222 / #18181b
- 次要文本：#45515e
- 品牌蓝：#1456f0（品牌）、#3b82f6（主色）
- 边框：#e5e7eb、#f2f3f5

### 3. 组件样式
- 按钮：主按钮 8px 圆角，导航使用全圆角（9999px）
- 卡片：20-24px 大圆角
- 阴影：品牌紫色调阴影 `rgba(44, 30, 116, 0.16)`

### 4. 排版规则
- 通用行高：1.50
- 标题默认字重：500-600（不用 700）
- 层级：Display 80px → Section 28-32px → Body 16px

### 5. 间距系统
- 基础单位 8px
- 大间距创造"画廊"效果

## 当前项目现状

- 使用 Tailwind CSS
- 已有自己的颜色和间距系统
- 组件使用圆角但不一致
- 使用 Lucide React 图标

## 目标

1. 更新 Tailwind 配置采用 MiniMax 色彩系统
2. 更新全局字体配置
3. 统一组件圆角和阴影
4. 优化整体视觉层次

## 非目标

- 不改变现有功能逻辑
- 不修改路由结构
- 保持 React 组件结构不变
